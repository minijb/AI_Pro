# Wiki Query — 查询与综合

在 wiki 中检索相关信息，综合回答用户问题。

---

## 触发条件

用户说：
- "关于 xxx 是什么/怎么样"
- "query"
- "wiki 中有没有提到"
- "帮我找找"
- 任何需要 wiki 内容支撑的问题

---

## 查询类型

| 类型 | 说明 | 处理方式 |
|------|------|---------|
| **事实查询** | X 是什么/怎么做 | 直接读相关页面回答 |
| **比较查询** | X 和 Y 有什么区别 | 收集双方信息对比 |
| **分析查询** | 分析 X 的原因/影响 | 多页面深度综合 |
| **探索查询** | X 和哪些概念相关 | 追踪链接图扩展回答 |
| **空白查询** | wiki 中没有 X 的内容 | 说明缺失，询问是否 ingest |

---

## 执行步骤

### 步骤 1：确定查询类型

根据问题措辞判断类型，选择对应处理策略。

### 步骤 2：定位相关页面

1. 读取 `wiki/index.md` 找到相关分类
2. 读取 `wiki/_index/index_by_category.md` 精确定位
3. 各分类索引页（`wiki/_index/index_XX_xxx.md`）内嵌了 Obsidian Bases 数据库视图，可交互式浏览该分类下所有笔记
4. 追踪 `[[wikilink]]` 链接找到相关内容

### 步骤 3：读取并综合

1. 读取所有相关页面的完整内容
2. **检查矛盾**：同一概念的冲突表述
   - 如有矛盾，标记 `⚠️ 矛盾：[[page-a]] vs [[page-b]]`
3. 构建有引用支撑的综合答案

### 步骤 4：呈现答案

使用引用标记：
```markdown
## {问题}

**答案**：
{综合内容}

**参考**：
- [[page-a]] — {为什么引用}
- [[page-b]] — {为什么引用}

**⚠️ 注意事项**（如有矛盾或过时）：
{说明}
```

### 步骤 5：归档优质答案（推荐）

如果答案具有以下特征，询问用户是否归档：
- 跨多个 wiki 页面的综合分析
- 对比分析
- 新洞察（不在现有 wiki 中）
- 常见问题的标准答案

归档方式：放入 `wiki/05_综合/` 文件夹
```markdown
# {综合标题}

---
title: {标题}
type: misc
tags: [{tag1}, {tag2}, 综合]
created: {YYYY-MM-DD}
related: [["page1"]], [["page2"]]
---

{答案内容}
```

### 步骤 6：更新 Memory/

如有新发现，更新：
- `Memory/relationships/SUMMARY.md`（新引用关系）
- `Memory/relationships/pages/{slug}.md`（具体页面引用关系）
- `Memory/focus_tracking.md`（当前查询的主题）

## 检索技巧

### 按分类检索

```
需要 X 语言相关 → 读取 wiki/01_语言/index.md
需要游戏引擎相关 → 读取 wiki/06_游戏开发/引擎/
需要 AI 相关 → 读取 wiki/04_领域/人工智能/
```

### 按时间检索

```
查找最近关注 → 读取 wiki/_index/index_by_time.md（含全部笔记的 Bases 按更新时间视图）
查找历史积累 → 读取 wiki/_index/index_by_time.md（更早 节）
```

### 追踪链接图

从 index 出发，沿 wikilink 扩展，直到覆盖所有相关内容。

### Bases 索引视图辅助定位

各分类索引页的 Bases 视图（`wiki/_index/base/index_XX_xxx_notes.base`）展示：
- **索引**（wikilink）、创建时间、更新时间、**描述**（一句话概括）
- 在 Obsidian 中打开 Bases 视图，可交互式浏览、筛选、排序该分类下所有笔记
- description 字段即笔记的一句话描述，可快速判断内容相关性

## 回答质量标准

1. **有引用**：每个要点都附上 `[[page]]` 引用
2. **标矛盾**：如有矛盾必须标记，不可忽略
3. **说空白**：wiki 中缺失的内容要说明
4. **提延伸**：给出 1-2 个延伸问题
