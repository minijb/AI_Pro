# Obsidian AI Wiki 系统对比报告

生成时间：2026-04-13

---

## 一、本系统概述

本系统是基于 **Obsidian + Claudaian 插件** 的 LLM Wiki 实现，采用 Karpathy LLM Wiki 模式，针对游戏开发者定制。

### 核心架构

| 层次 | 说明 |
|------|------|
| **raw/** | 源文档（归档后不可变） |
| **wiki/** | LLM 全权维护的知识库 |
| **Memory/** | 跨会话上下文追踪 |

### 五大核心操作

| 操作 | 触发词 | 功能 |
|------|--------|------|
| **ingest** | "处理这个源"/"ingest" | 读取源 → 创建 wiki 页面 → 更新索引 |
| **query** | "关于 xxx"/"query" | 搜索 wiki → 综合答案 → 归档优质答案 |
| **lint** | "整理 wiki"/"lint" | 健康检查：矛盾/过时/孤立页面 |
| **process** | "整理一下"/"精炼" | 精炼/合并/拆分/迁移内容 |
| **analyze** | "分析这个"/"文档分析" | 深度分析源文档 → 生成报告 |

### 特色机制

- **矛盾主动标记**：`> [!warning] 矛盾`
- **四层索引**：主索引 → 分类索引 → Bases 数据库视图 → 时间索引
- **归档不可变**：raw/ 归档后绝不修改
- **Memory 分片**：按功能分割（stats/、relationships/、focus_tracking/）

## 二、其他 Obsidian AI 系统

### 2.1 Claudian（本系统所使用）

- **定位**：将 Claude Code 作为 AI 协作者嵌入保险库
- **特点**：与本系统深度整合，通过 Claudaian 插件实现

### 2.2 Obsidian Copilot

- **定位**：AI 写作辅助
- **特点**：
  - 内联 AI 补全
  - 写作建议
  - 嵌入式 AI 对话
- **局限**：主要是写作辅助，无系统性知识管理

### 2.3 Smart Connections

- **定位**：AI 语义搜索
- **特点**：
  - 支持多模型（Claude、Gemini、ChatGPT、Llama 3）
  - 向量嵌入检索
  - 自然语言查询笔记
- **局限**：仅搜索，无 ingest/lint 流程

### 2.4 Text Generator

- **定位**：文本生成
- **特点**：
  - 支持多 AI 提供商（OpenAI、Anthropic、Google、本地模型）
  - 模板化生成
- **局限**：单点生成工具，非系统性知识管理

### 2.5 Dataview

- **定位**：数据索引和查询
- **特点**：
  - 类 SQL 查询语言
  - 动态数据视图
- **局限**：查询工具，非 AI 驱动

### 2.6 Templater

- **定位**：模板自动化
- **特点**：高级模板语法
- **局限**：模板工具，非知识管理

## 四、差异分析

### 4.1 本系统独有的功能

| 功能 | 说明 |
|------|------|
| **矛盾主动标记** | 唯一主动检测并标记知识矛盾的系统 |
| **五操作完整流程** | ingest → query → lint → process → analyze 闭环 |
| **Memory 分片追踪** | stats/、relationships/、focus_tracking/ 跨会话记忆 |
| **归档不可变原则** | raw/ 归档后绝不修改，保证知识溯源 |
| **游戏开发者定制** | 8 类分类体系针对游戏开发优化 |

### 4.2 本系统可优化的地方

#### 高优先级

| 问题 | 当前状态 | 优化建议 |
|------|----------|----------|
| **单一模型限制** | 仅使用 haiku 模型 | 支持模型切换（GPT-4/Claude Sonnet），尤其在 ingest/query 时 |
| **无向量嵌入** | 纯文本匹配 | 集成 embedding 提升语义搜索能力 |
| **无 Web Search** | 仅本地 wiki | 添加 Web Search 工具，ingest 时可补充外部信息 |

#### 中优先级

| 问题 | 当前状态 | 优化建议 |
|------|----------|----------|
| **无工具调用** | 手动触发各操作 | 探索 Claude Code 工具调用能力 |
| **无增量更新** | 每次全量更新索引 | 实现增量索引更新机制 |
| **无 AI 补全** | 无内联写作辅助 | 考虑集成 Copilot 的写作补全能力 |

#### 低优先级

| 问题 | 当前状态 | 优化建议 |
|------|----------|----------|
| **无多用户支持** | 单用户 | 暂无需求，可忽略 |
| **无移动端** | 仅桌面 | 暂无需求，可忽略 |
| **无实时协作** | 异步工作流 | 暂无需求，可忽略 |

## 五、优化建议详述

### 5.1 最高优先级：模型升级

**问题**：haiku 模型能力有限，影响 ingest/query 质量。

**方案**：
```json
// .claude/settings.local.json
{
  "model": "sonnet",  // ingest/query 时使用
  "haikuModel": "haiku"  // 简单操作时使用
}
```

**预期收益**：
- ingest 时更好地提取关键要点
- query 时更准确综合答案
- 更强的矛盾检测能力

### 5.2 次优先级：向量搜索集成

**问题**：当前纯文本匹配，无法理解语义相似性。

**方案**：
1. 添加 Text Generator 或 Smart Connections 作为补充搜索
2. 或在 query 时调用外部 embedding API

**预期收益**：
- 语义相近但措辞不同的问题能被正确关联
- 发现隐含关联的知识

### 5.3 第三优先级：Web Search 集成

**问题**：ingest 时只能基于已有 wiki 内容，缺乏外部信息。

**方案**：
1. 在 ingest 的"与用户讨论要点"环节添加 Web Search
2. query 时可搜索外部资源补充

**预期收益**：
- 摄入时补充最新信息
- query 时提供更全面的答案

## 六、总结

| 维度 | 评分 | 说明 |
|------|------|------|
| 系统性 | ⭐⭐⭐⭐⭐ | 五操作完整闭环，其他系统无此完整性 |
| 知识管理 | ⭐⭐⭐⭐⭐ | 矛盾标记、交叉引用、Memory 追踪业界领先 |
| AI 能力 | ⭐⭐ | 单一模型、无向量嵌入、无工具调用 |
| 可扩展性 | ⭐⭐⭐ | 架构清晰，但缺少向量/搜索集成 |

**核心优势**：本系统的最大优势是**系统性的知识管理流程**和**矛盾主动检测机制**，这是其他 Obsidian AI 插件所不具备的。

**主要短板**：AI 能力受限于单一模型，缺少向量搜索和 Web Search 集成。

**建议行动**：
1. 优先升级模型配置（高优先级）
2. 考虑添加向量搜索能力（中优先级）
3. 探索 Web Search 工具调用（中优先级）
