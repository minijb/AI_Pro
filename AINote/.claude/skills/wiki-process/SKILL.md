# Wiki Process — 处理与精炼

对已摄入的内容进行处理、精炼、合并或重构。

---

## 触发条件

用户说：
- "整理一下"
- "处理这些内容"
- "精炼"
- "合并这两个页面"
- 提供了需要处理的 wiki 页面列表

---

## 处理类型

| 类型 | 说明 |
|------|------|
| **精炼** | 提要点、删冗余、通俗化表述 |
| **合并** | 将多个相关页面合并为一个 |
| **拆分** | 将一个大页面拆分为多个子页面 |
| **迁移** | 将内容从一个分类移动到另一个 |
| **更新** | 用新信息更新已有页面的过时内容 |

---

## 执行步骤

### 精炼（Refine）

1. 读取目标页面的完整内容
2. 识别：
   - 核心信息（保留）
   - 冗余表述（删除或简化）
   - 缺失的重要信息（补充）
3. 改写为更简洁、更易引用的格式
4. 更新 frontmatter 的 `updated` 日期

### 合并（Merge）

1. 读取所有要合并的页面
2. 确定合并策略：
   - **新页合并**：创建全新页面，整合所有内容
   - **追加合并**：选定基础页，其他页内容追加进去
3. 识别并解决冲突（同一概念的不同表述）
4. 删除被合并的旧页面
5. 更新所有指向旧页面的 wikilink
6. 更新 `Memory/relationships.md`

### 拆分（Split）

1. 读取要拆分的大页面
2. 按主题/时间/功能拆分为多个子页面
3. 创建父级概览页，引用各子页面
4. 更新 `Memory/relationships.md`

### 迁移（Move）

1. 确定源位置和目标位置
2. 移动文件
3. 扫描所有 wiki 页面，更新引用
4. 更新 `wiki/index.md`、`wiki/_index/index_by_category.md`、`wiki/_index/index_by_time.md`（分类索引页的 Bases 视图从 frontmatter 自动读取，无需手动更新表格）

### 更新（Update）

1. 读取已有页面
2. 与用户提供的新信息对比
3. 标记过时内容（添加 `⚠️ 已过时：` 前缀）
4. 补充新内容
5. 更新 `updated` 日期
6. 如有矛盾，标记到 `Memory/relationships.md`

---

## 交叉操作规则

- **合并/拆分/迁移后必须更新**：
  - `wiki/index.md`
  - `wiki/_index/index_by_category.md`、`wiki/_index/index_by_time.md`
  - `wiki/log.md`（追加操作记录）
  - `Memory/relationships/`（更新相关页面的引用关系）
  - `Memory/stats/`（更新受影响子目录的状态文件）

---

```

变化：

涉及页面：
- {page1}
- {page2}

注意事项：
- {任何需要用户知晓的信息}
```
